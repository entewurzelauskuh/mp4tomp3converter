# Contributing

Thanks for your interest! This is a deliberately small, single-purpose app — please read
`MP4toMP3converter-SPEC.md` (the source of truth) and `docs/DECISIONS.md` before proposing
changes, and keep the scope narrow (see `docs/ROADMAP.md` for what's intentionally out of scope).

## Setup

- macOS build host, JDK 21, Android SDK. Create `local.properties` with `sdk.dir=…`.
- Native build needs the NDK + CMake (versions pinned in `app/build.gradle.kts`).
- `third_party/lame/` is **vendored, unmodified** LAME source — never edit it.

## Before you commit

```sh
./gradlew spotlessApply test        # format + fast JVM unit tests (spec convention)
```

Full checks (see `docs/TESTING.md` for the emulator setup — **emulator only**, never a physical
device):

```sh
./gradlew test connectedDebugAndroidTest lint spotlessCheck assembleRelease
```

## Conventions

- **Kotlin only** (Java only for unavoidable JNI glue). Gradle Kotlin DSL + the version catalog
  (`gradle/libs.versions.toml`) — no hardcoded dependency versions.
- **All user-facing strings** go in `res/values/strings.xml` — no hardcoded UI strings.
- **Never** add the `INTERNET` permission or any dependency not justified by the spec. A
  build-time check enforces the no-INTERNET rule.
- ktlint via Spotless; Kotlin official style. KDoc on public types; comments explain *why*.
- Do **not** change the frozen contracts in spec §6.2 (`ConversionJob`, `JobState`,
  `AudioConverter`, `OutputSink`, `JobRepository`) without updating the spec first.
- Conventional-commit-style messages; every change should build and keep tests green.

## Tests

- JVM unit tests: `jobs/` (queue state machine, drain loop), `output/FileNaming`,
  `engine/ProgressThrottler`, settings serialization.
- Instrumented (emulator, API 31 and 34/35): engine conversion, sinks, Compose UI, and the
  end-to-end MediaStore playback test. Fixtures are generated by `scripts/make_fixtures.sh`
  (needs `brew install ffmpeg` — a dev tool only, never shipped).
